## 2. 배경 및 목적

스승의 날(5월 15일)을 맞아 정보 교과 CCL 단원 수업에서, 각 반 학생들이 담임 선생님 몰래 MS Forms를 통해 ① 삼행시, ② 담임 선생님 매력 포인트, ③ 감사 편지, ④ 선생님 그리기, ⑤ CCL 라이선스 선택을 제출했다.

담당 정보 선생님의 사정으로 배송이 **한 달 지연**되었으나, 이를 오히려 **"배송 지연" 유머 컨셉**으로 승화하여 카드뉴스 웹페이지로 제작한다.

작년에는 Canva로 수작업 제작했으나, 올해는 **Python 자동화 (vibe coding)** 으로 1-1반 ~ 1-8반 8개를 일괄 생성한다.

## 7. Claude API 분석 명세

생성 시점에 1회 실행 후 `analysis/1-X_analysis.json`에 저장, HTML에 주입:

| 분석 작업 | 입력 | 출력 JSON 형식 |
|-----------|------|----------------|
| 삼행시 연결 스토리 | 전체 삼행시 행 | `{ "story": "string" }` |
| 감정 워드클라우드 | 편지 + 매력포인트 전체 | `[{ "word": "", "size": 1~5, "category": "감사/애정/존경/기쁨/안정감" }]` |
| 능력 카테고리 분류 | 매력포인트 전체 | `{ "공감": [], "관계": [], "정서": [], "매력": [] }` |
| 영향력 분석 | 편지 전체 | `[{ "title": "", "desc": "" }]` × 4항목 |
| 마인드맵 브랜치 | 편지 전체 | `{ "감사함": [], "사랑함": [], "존경심": [], "기대감": [], "성장의지": [] }` |
| 그림 공통 요소 | 학생 그림 이미지 파일들 | `["string"]` × 6개 불릿 |

## 8. Firebase 설계

### 8-1. Firestore 컬렉션 구조

```
replies/                         ← 컬렉션
  └── 1-1                        ← 문서 ID = 반 번호
        teacher_name: "이서영"
        reply_text:   "얘들아, 선생님도 너희가 정말 좋아 💕"
        replied_at:   2026-06-15T10:30:00Z
        is_published: true        ← false면 학생에게 미노출
```

### 8-2. 보안 규칙 (Firestore Rules)

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /replies/{classId} {
      // 누구나 읽기 가능 (링크 아는 사람)
      allow read: if true;
      // 쓰기는 Firebase Auth 로그인한 사람만
      allow write: if request.auth != null;
    }
  }
}
```

### 8-3. 인증 방식

- **Firebase Google 로그인** — 담임 선생님이 Google 계정으로 인증 후 답장 작성
- 학생들은 로그인 없이 읽기만 가능
- 로그인 버튼은 에필로그 슬라이드에만 표시
